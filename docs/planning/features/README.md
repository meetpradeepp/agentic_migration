# Feature-Level Implementation Plans

This directory contains feature-level implementation plans generated by the Planner Agent.

## Directory Structure

Each feature gets its own directory:

```
features/
├── feature-1-joern-ingestion/
│   ├── spec.md                   # Feature specification (input)
│   ├── implementation_plan.json  # Planner Agent output
│   ├── init.sh                   # Setup script (optional)
│   ├── build-progress.txt        # Progress tracking (optional)
│   └── subtasks/                 # Subtask tracking (optional)
│       ├── subtask-1.md
│       └── subtask-2.md
│
└── feature-6-proc-extraction/
    ├── spec.md
    ├── implementation_plan.json
    └── ...
```

## Workflow

1. **User** selects feature from `docs/roadmap/roadmap.json`
2. **Orchestrator** invokes **Planner Agent** for that feature
3. **Planner Agent** creates `docs/planning/features/[feature-slug]/implementation_plan.json`
4. **Implementation Agent** (future) executes subtasks from implementation_plan.json

## Naming Convention

Feature directory names should use kebab-case slugs derived from the feature ID or name:
- `feature-1-joern-ingestion` (from feature-1: "Joern Code Ingestion")
- `feature-6-proc-extraction` (from feature-6: "Pro*C Embedded SQL Extraction")
- `user-authentication` (direct feature name)

See `.github/instructions/planner.instructions.md` for planning guidelines.
